What is GeoMunchers?

The goal of the game is for the player to eat all the munchables on the board. Munchables are shapes/angles and the objectives (which shape to munch) are randomized.

Game Object


import pygame
import random

class GameObject:
    def __init__(self, board, boardX, boardY):
        self.board = board
        self.screen = board.surface

        self.screenRect = {'x':0, 'y':0, 'width':0, 'height':0}
        self.boardCoords = {'x':boardX, 'y':boardY}

        #More primitive datastructure for blitting to screen
        self.drawRect = None

from gameobject.GameObject import Munchable

class Level:

    Levels = []

    def __init__(self, screen, levelName, goodMunchableTypes, badMunchableTypes, numOfGoodMunchables):
        self.screen = screen;

        self.levelName = levelName

        self.goodMunchableTypes = goodMunchableTypes
        self.badMunchableTypes = badMunchableTypes
        self.numOfGoodMunchables = numOfGoodMunchables

        Level.Levels.append(self)

Challenges we had

Challenges with code

  • Platform dependence(Windows doesn't support GDK3)
  • Python learning curve -> applying it to a OO game programming
  • Time and scope
  • Speghetti code

Challenge with art

  • Animating a player
  • Choosing an appropriate compression algorithm

Challenge with documentation

  • Wiki markup language has a high learning curve
  • Using new tools to make pretty documentation

What we would have done differently

  • Invested more time in the project - more features implemented
  • Dividing work better and making sure all did their share
  • More accurately scoped out project
